# Astro 6.4 Release

**Source:** [Astro Blog](https://astro.build/blog/astro-640/)  
**Date:** 2026-05-29

---

## TL;DR

Pluggable Markdown pipeline, experimental Rust-based processor (Sätteri), and Cloudflare helpers for advanced routing.

---

## New `markdown.processor` API

Markdown pipeline is now swappable. Default remains `unified()`, but you configure `remark`/`rehype` plugins on the processor directly:

```javascript
import { defineConfig } from 'astro/config';
import { unified } from '@astrojs/markdown-remark';
import remarkToc from 'remark-toc';

export default defineConfig({
  markdown: {
    processor: unified({
      remarkPlugins: [remarkToc],
    }),
  },
});
```

### Deprecated (removed in Astro 8.0)
- `markdown.remarkPlugins`
- `markdown.rehypePlugins`
- `markdown.remarkRehype`
- `markdown.gfm`
- `markdown.smartypants`

Migrate to `processor: unified({...})` format.

## Sätteri — Rust-Based Markdown Processor

Experimental `@astrojs/markdown-satteri` package. Powered by [Sätteri](https://satteri.bruits.org/).

- **Significantly faster** than default `unified` pipeline
- **Native feature support** for many Markdown features (no plugins needed)
- **Performance:** Astro + Cloudflare docs sites saved **>1 minute** each in build time
- **Limitation:** No `remark`/`rehype` plugin support. Port to Sätteri MDAST/HAST plugins if needed
- **Future:** Likely becomes default in a future major version

```bash
npm install @astrojs/markdown-satteri
```

```javascript
import { satteri } from '@astrojs/markdown-satteri';

export default defineConfig({
  markdown: {
    processor: satteri({
      features: { directive: true },
    }),
  },
});
```

See [Native Markdown / MDX RFC](https://github.com/withastro/roadmap/pull/1364).

## Cloudflare Helpers for Advanced Routing

For experimental advanced routing (Astro 6.3+), `@astrojs/cloudflare` now provides a `cf()` helper configuring:
- SESSION KV binding injection
- Static asset serving via ASSETS binding
- `locals.cfContext`
- Client address from `cf-connecting-ip`
- `waitUntil`
- Prerendered error pages

### Custom Fetch Handler

```javascript
import { astro, FetchState } from 'astro/fetch';
import { cf } from '@astrojs/cloudflare/fetch';

export default {
  async fetch(request, env, ctx) {
    const state = new FetchState(request);
    const asset = await cf(state, env, ctx);
    if (asset) return asset;
    return astro(state);
  },
};
```

### Hono Middleware

```javascript
import { Hono } from 'hono';
import { actions, middleware, pages, i18n } from 'astro/hono';
import { cf } from '@astrojs/cloudflare/hono';

const app = new Hono<{ Bindings: Env }>();

app.use(cf());
app.use(actions());
app.use(middleware());
app.use(pages());
app.use(i18n());

export default app;
```
